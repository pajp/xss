Bricole XML Session Server

The XSS Configuration-Based Authentication Module

Overview

Full name: se.bricole.xss.modules.ConfigBasedAuthenticator
Description: The Configuration-Based Authentication Module provides an easy way enable to authentication for your modules. User data is entered as module properties for this module in the regular XSS configuration file.
It supports both basic (plain text) and challenge-response authentication. The authentication process is described in the generic XML module's "<auth/>" tag documentation

Configuration

Following is an example of the XSS configuration directive relative to an instance of this module. 


                <Module source="classpath" name="se.bricole.xss.modules.ConfigBasedAuthenticator">
                    <Property name="domains" value="xss.system" />
                    <Property name="password.admin" value="xssAdmin" />
                </Module>

The domains property specifies which authentication domains this instance of the module should handle. Each instance can be responsible for an arbitrary number of domains, each specified in this property as a comma-separeted list. Example: <Property name="domains" value="system,chat"> configures the module to handle the domains "system" and "chat".

The other properties specifies the passwords for the user. Each password is specified using the property name password.username where username is substituted with the user the password belongs to. Example: <Property name="password.rasmus" value="joshua1"> configures a user "rasmus" with the password "joshua1".
